Visão Geral do Sistema
Sistema: Quanta Shop — Plataforma de Cashback e Afiliados

URL de produção: quantashop.com.br — Plataforma B2C/B2B conectando consumidores, lojistas e afiliados por cashback automático e rede de indicações MLM.

Stack Atual vs. Recomendação Lovable

Stack Atual (Nuxt 3)
  • Framework: Nuxt 3 + Vue 3 + TypeScript
  • CSS: Bootstrap 5 + SCSS
  • Estado: Pinia
  • Renderização: SPA (SSR desabilitado)
  • Auth: JWT em localStorage['agencia_user']
  • Backend: .NET 8, porta 8000, Azure SQL
  • Proxy: Nitro /api-proxy/*
Stack Recomendada (Lovable)
  • Framework: React 18 + TypeScript
  • CSS: Tailwind CSS
  • Roteamento: React Router v6
  • Estado: Zustand
  • HTTP: Axios + interceptor JWT
  • Formulários: React Hook Form + Zod
  • Tabelas: TanStack Table
  • Gráficos: Recharts
  • UI: Headless UI / Radix UI
IMPORTANTE — Regras de Integração com Replit
  • Usar SEMPRE paths relativos /api-proxy/* — nunca URLs absolutas
  • Manter chave localStorage agencia_user (compatibilidade com código existente)
  • NÃO alterar estrutura de rotas da API .NET no backend Replit

/agencia/login — Login do Afiliado

  • Campos: Login ou e-mail, Senha (toggle visibilidade)

/agencia/cadastro — Cadastro de Afiliado

  • Campos: nome completo, login (único), e-mail, senha, confirmação, CPF (máscara), telefone (máscara), indicador (query param pré-preenchido)
  • Ação: POST /usuario/cadastrar
  • Validação: CPF válido, e-mail único, senha ≥ 8 chars, confirmação igual

Hook useApi — Axios configurado para o proxy Replit

Criar src/lib/api.ts: instância Axios com baseURL: '/api-proxy' e interceptor que lê localStorage.getItem('agencia_user'), extrai o token e adiciona o header Authorization: Bearer TOKEN em toda requisição. Interceptor de resposta: redirecionar para /login se HTTP 401.

Fluxo de Autenticação JWT
Fluxo completo — 7 passos
  1. Usuário submete login → POST /usuario/autenticar com body {"login": "...", "senha": "..."}
  2. API retorna JSON com token (JWT), nome, login, perfil, e demais dados do usuário
  3. Frontend armazena: localStorage.setItem('agencia_user', JSON.stringify(responseData))
  4. Toda requisição autenticada: Header Authorization: Bearer {token} via interceptor Axios
  5. Guard de rota: Antes de renderizar página protegida, verificar se agencia_user existe e decodificar JWT para checar campo exp (expiração)
  6. Token expirado: localStorage.removeItem('agencia_user') → redirect para /agencia/login
  7. Perfil admin: Verificar campo perfil no payload JWT — se não for admin, bloquear /admin/* e redirecionar

Guia de Integração Lovable → Replit
Regras Inegociáveis
  • Usar SEMPRE paths relativos /api-proxy/* — nunca https://api.quantashop...
  • Manter chave agencia_user em localStorage
  • Não alterar rotas da API .NET no Replit backend

O Proxy Nitro (já configurado)

O arquivo server/routes/api-proxy/[...path].ts recebe /api-proxy/ENDPOINT e faz proxy para http://localhost:8000/ENDPOINT, passando todos os headers (incluindo Authorization). CORS já está resolvido nesta camada.
