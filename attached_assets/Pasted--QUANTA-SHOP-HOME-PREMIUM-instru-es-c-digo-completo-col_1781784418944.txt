# QUANTA SHOP — HOME PREMIUM: instruções + código completo (colar no Agent do Replit)

INSTRUÇÕES PARA O AGENT (leia tudo antes de começar):

Este documento contém o CÓDIGO COMPLETO de 20 arquivos da nova home premium do site Quanta Shop
(Nuxt 3 SPA, Vue 3 <script setup lang="ts">, CSS scoped, design system qs-*, SEM Bootstrap/Tailwind).
Crie/atualize cada arquivo no caminho EXATO indicado em cada bloco "INÍCIO ARQUIVO: <caminho>".
O conteúdo do arquivo é tudo que está entre a linha "INÍCIO ARQUIVO" e a linha "FIM ARQUIVO".

REGRAS:
1) Crie os arquivos novos e SUBSTITUA os existentes pelos conteúdos abaixo — COM UMA EXCEÇÃO:
   - "composables/useHomeConfig.ts": se o projeto JÁ tiver esse arquivo, MANTENHA o existente (não troque).
     O incluído aqui é só um fallback; o hero funciona sem ele (usa 6 banners default internos).
2) O arquivo "pages/index.vue" é o que faz a home nova aparecer — ele monta os 13 componentes home-*.
   Substitua o index atual por este (preservando rotas/SEO que por acaso já existam e sejam necessárias).
3) Respeite as restrições da plataforma: ssr:false, porta 5000, allowedHosts:true; NUNCA usar "localhost"
   hardcoded; não introduzir Bootstrap/Tailwind; não usar base64 em imagens.
4) Imagens: este documento é só TEXTO, então as imagens NÃO vêm aqui. Os componentes apontam para:
   /img/logo/quanta-logo.png, /img/logo/quanta-logo-whitegreen.png, /img/logo/quanta-icon.png,
   /img/logo/quanta-icon-white.png, /img/team/mauro-cutout.png, /og-cover.jpg, /favicon.png
   O usuário vai subir esses arquivos de imagem na pasta public/ depois (arrastando). Se algum ainda não
   existir, NÃO quebre o build — apenas deixe a referência (a imagem aparece quando for enviada).
5) Ao terminar, rode: npm run build:nuxt  (0 erros; corrija imports se necessário).
6) Confirme na home: hero escuro com 6 banners girando + chat "Quanta IA"; seção "Como funciona";
   seção do CEO com a foto; rodapé com logo branca+verde; menu com "Como funciona" como 1º item;
   e a rota /como-funciona funcionando. Teste no mobile (largura ~390px) sem rolagem horizontal.

 * `loadConfig` com o fetch — os componentes já leem `config.value.<chave>`
 * com fallback para os arrays default, então nada mais precisa mudar.
 */
export interface HomeConfig {
  heroBanners?: unknown[];
  [key: string]: unknown;
}

let loaded = false;

  async function loadConfig(): Promise<void> {

    loaded = true;

    // try { config.value = await $fetch('/api/home-config'); } catch { /* usa defaults */ }
  }
  return { config, loadConfig };
}

import { computed } from 'vue';

const props = withDefaults(defineProps<{
  /** 'icon' = só o símbolo bq · 'full' = logo + wordmark "quanta SHOP" */
  variant?: 'icon' | 'full';
  /** 'color' = teal+lime · 'white' = monocromático p/ fundo escuro (só variant icon) */
  tone?: 'color' | 'white';
  /** altura em px */
  size?: number;
  /** se passado, vira <nuxt-link> */
  to?: string;
}>(), { variant: 'icon', tone: 'color', size: 40 });

const alt = 'Quanta Shop';
const src = computed(() => {
  if (props.variant === 'full') return '/img/logo/quanta-logo.png';
  return props.tone === 'white' ? '/img/logo/quanta-icon-white.png' : '/img/logo/quanta-icon.png';
});
</script>
